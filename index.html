const stack = ['HTML', 'CSS', 'TypeScript', 'Node.js'];

type SprintPhase = 'prep' | 'intensive' | 'practice';

interface Module {
  title: string;
  weeks: number;
  focus: string[];
}

const roadmap: Record<SprintPhase, Module> = {
  prep: {
    title: 'Online-підготовка з ментором',
    weeks: 4,
    focus: ['основи JS', 'Git', 'робочий процес']
  },
  intensive: {
    title: 'Full-time занурення',
    weeks: 14,
    focus: ['frontend', 'backend', 'архітектура']
  },
  practice: {
    title: 'Вечірня практика',
    weeks: 6,
    focus: ['командні проєкти', 'code review']
  }
};

function startSprint() {
  return roadmap.intensive.focus.includes('frontend')
    && roadmap.intensive.focus.includes('backend');
}

Хто такий Fullstack

Хто такий fullstack-розробник?

Fullstack-інженер — універсальний IT-фахівець, який працює з frontend і backend. Він здатен спроєктувати та реалізувати повноцінний веб-додаток — від інтерфейсу до бази даних і деплою.

Спрос на таких спеціалістів стабільно перевищує пропозицію: компаніям потрібні люди, які розуміють продукт цілісно, можуть спілкуватися з різними командами та закривати вузькі місця як на боці клієнта, так і на боці сервера.

Методика

Про методику Sprint

Ми поєднуємо інтенсивний формат, чітку структуру та постійний фідбек, щоб максимально прискорити шлях від нуля до першої роботи.

Що це?

Sprint-підхід — це формат повного занурення з акцентом на практику. Кожен тиждень — це міні-спринт із конкретною ціллю, дедлайнами та результатом, який можна показати в портфоліо.

Чому це ефективно?

Висока щільність задач, постійна практика, командна робота й контроль прогресу дозволяють швидко накопичувати досвід. Ви не «просто дивитеся уроки», а щодня пишете код, отримуєте фідбек і покращуєте свої рішення.

Де це застосовується?

Такий підхід використовується в технологічних компаніях і стартапах: команди працюють короткими ітераціями, швидко тестують гіпотези й регулярно відправляють фічі в продакшн. Ми відтворюємо цей підхід у навчанні.
